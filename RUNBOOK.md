# Epistemic ARC Validation Runbook

This runbook outlines the strict, reproducible steps to bring an H100 (SM90a) pod from a clean state to a fully executing validation sweep, avoiding all hand-jammed environment setup errors.

## 1. Local Pre-Flight (On Mac)

Before provisioning the pod or immediately after SSH access:

```bash
# Push latest code from local to GitHub
git push origin main

# Upload the base dataset directly to the pod
scp -r '/Users/oli/Documents/Oli Works/arc_epistemic/Kaggle_Submission_Bundle/dataset/' root@<POD_IP>:/workspace/arc_dataset/
```

## 2. Pod Initialization (On Server)

Clone the repository and run the idempotent restorer script. This handles system packages, CUDA compilation, venv creation, and model downloading.

```bash
# 1. Clone your repo
git clone https://github.com/Steake/Sovereign-Epistemic-Agent.git /workspace/Sovereign-Epistemic-Agent

# 2. Fire the master setup script
# This builds everything natively for H100 and pulls Qwen3 weights.
HF_TOKEN=hf_... bash /workspace/Sovereign-Epistemic-Agent/scripts/restore_pod.sh --dataset-dir /workspace/arc_dataset
```

## 3. Launching the Inference Engine

`restore_pod.sh` attempts to ignite the server, but for recovery configurations where you want to alter the context block or reasoning budget, use the pre-built `ignite_recovery.sh` wrapper:

```bash
# Example: Launch with Reasoning disabled
bash /workspace/Sovereign-Epistemic-Agent/scripts/ignite_recovery.sh 0

# Example: Launch with 512-token reasoning budget
bash /workspace/Sovereign-Epistemic-Agent/scripts/ignite_recovery.sh 512
```

## 4. Health Check & Sanity Request

Ensure the server is actually responding and respects the parameters:

```bash
# Active wait for server health
curl -sf http://localhost:8000/health

# Sanity curl: ensure the model parses simple JSON commands
curl -s http://localhost:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"Qwen3.5-27B.Q8_0.gguf","messages":[{"role":"user","content":"Output ONLY: {\"sanity\":\"check\"}"}],"max_tokens":200,"temperature":0.1}'
```

## 5. Kickoff Validation

Load the initialized environment variables generated by the setup wrapper, and execute the benchmark.

```bash
# Source the exact Python/DataSet environment paths
source /workspace/.container_env

# SMOKE TEST (Run the 8-task fast recovery suite)
# Make sure to edit scripts/full_validation.py to point `manifest_path` to `data/smoke_test_manifest.txt` momentarily if testing.
nohup python3 /workspace/Sovereign-Epistemic-Agent/scripts/full_validation.py > /workspace/sweep.log 2>&1 &
echo "Sweep PID: $!"

# FULL RUN (50-task suite)
nohup python3 /workspace/Sovereign-Epistemic-Agent/scripts/full_validation.py > /workspace/sweep.log 2>&1 &
echo "Sweep PID: $!"
```

## 6. Real-Time Telemetry

```bash
# Watch the sweep logs unfold actively:
watch -n 30 "grep 'Ground-truth match:' /workspace/sweep.log | wc -l && grep 'Ground-truth match: True' /workspace/sweep.log | wc -l && tail -15 /workspace/sweep.log"
```
